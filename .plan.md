# 进攻/受击热力图实现计划

## Context

用户希望基于 `events` 表中的 `发弹`（射击位置）和 `受击`（受伤位置）两类事件，为每个学校各生成两张热力图：**进攻热力图**和**受击热力图**。需要修改导出脚本生成数据，修改前端 `analysis.html` 支持切换显示。

## 数据来源

- `events` 表有 `事件类型`（发弹/受击）、`robot_id`、`时刻秒` 等字段，但**没有 x/y 坐标**
- 需要 JOIN `timeseries` 表（`game_id + robot_id + 时刻秒`）获取事件发生时的位置
- events 表有约 3x 重复行，需用 DISTINCT 去重

## 实现步骤

### 1. 修改 `scripts/export_position_data.py` — 新增导出函数

新增 `export_event_heatmap(conn)` 函数：
- 查询发弹事件：`SELECT DISTINCT 学校名, 阵营, t.x, t.y FROM events e JOIN timeseries t ON ... WHERE e.事件类型='发弹'`
- 查询受击事件：同上，`WHERE e.事件类型='受击'`
- 按学校、阵营（红/蓝）分别构建 60x60 网格
- 归一化到 0-100，base64 编码
- 输出格式与现有 position heatmap 一致：`{red: base64, blue: base64}`

在 `main()` 中调用，生成独立文件：
- `static/heatmap/{region}/{school}_attack.json` — 进攻热力图
- `static/heatmap/{region}/{school}_damage.json` — 受击热力图

同时更新 `config.json`，添加 `heatmapTypes: ["position", "attack", "damage"]` 字段。

### 2. 修改 `analysis.html` — 前端支持热力图类型切换

- 修改页面标题和 hero 文字为"热力图"
- 修改导航链接文字为"热力图"
- 在筛选栏添加"类型"按钮组：`位置热力图 | 进攻热力图 | 受击热力图`
- 新增全局状态 `currentHeatmapType`（'position' / 'attack' / 'damage'）
- 修改 `loadSchoolData()` 根据类型加载不同文件（`{school}.json` / `{school}_attack.json` / `{school}_damage.json`）
- 修改 `renderHeatmap()` 根据类型选择对应数据 key 渲染
- 位置热力图保留兵种筛选，进攻/受击热力图隐藏兵种筛选（因为是所有兵种合并的事件数据）
